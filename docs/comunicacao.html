Gerenciamento de Comunicações

Overview

As ferramentas escolhidas pela equipe buscam ao máximo reduzir falhas de comunicação, bem como como coletar, armazenar e distribuir informações dentro do projeto.

Ferramentas

Telegram
  • Para comunicação instantânea entre membros da equipe;
  • Para realização de dailies, pontualmente às 16h; (Auxílio de Bot)
  • Para notificar membros de abertura/fechamento de issues ou pull requests, e qualquer outro nível de interação no repositório/issue; (Auxílio de Bot);
Google Drive
  • Para escrever e versionar documentos;
  • Para montar apresentações;
GitHub
  • Para armazenar e versionar código;
  • Uso de issues para rastrear e delegar tarefas;
  • Uso da seção de comentários em issues para exposição de dúvidas e materiais, centralizando e facilitando o acesso de todos os membros da equipe.
  • Para, através do github pages, apresentar as features do projeto ao público;
ZenHub
  • Para monitorar a produtividade da equipe através do burndown, e do velocity;
  • Para rastrear o andamento das histórias através do kanban;

Rituais

Reuniões semanais aos sábados, todas timeboxed, para realização dos rituais scrum, sendo eles:

  • Review - Uma hora;
  • Retrospectiva - Duas horas
    • (Sofreu ajuste para uma hora);
  • Planejamento - Uma hora e quarenta e cinco minutos;
